📖 公式マニュアル & ヘルプ

ショートカット一覧

🏞️ 基本・ステージ編集

  • G : 移動 (Translate)
  • R : 回転 (Rotate)
  • S : 拡大 (Scale)
  • Ctrl + D : オブジェクト複製
  • Del / BS : 削除
  • Esc : 選択解除 / プレイ停止
  • Ctrl + Z / Y : Undo / Redo

🤖 アニメーション

  • Space : 再生 / 停止
  • ← / → : 1フレーム戻る / 進む
  • Shift + ← / → : 最初 / 最後へ飛ぶ
  • I : キーフレーム挿入 (Insert)
  • Ctrl + Wheel : タイムラインのズーム

※入力欄（名前や数値など）に文字を打っている間はショートカットは無効になります。

🏞️ Stage Design (ステージ作り)

画面下のパレットから図形を追加し、世界を作りましょう。
右側の「プロパティ」で、色や物理挙動、さらに「ゲームロジック（役割）」を設定できます。

  • 物理挙動: 床や壁は「Static(不動)」、動くリフトは「Kinematic(制御)」にします。
  • スタートとゴール: 最低1つずつ配置しないとゲームが始まりません。
  • ワープとポータル: 入口と出口のIDを合わせることで一瞬で移動できます。
  • イベント発火: 透明にして配置すると、踏んだ時に「チュートリアルUI」を開くなどの罠が作れます。
  • ランダムスポナー: 指定した範囲に、敵をまとめてランダム配置できます。

🤖 Character Design (キャラ・AI作り)

敵キャラやNPCをモデリングし、動きと性格を与えます。

  • モデリング: パーツの親を「Root」や「Body」に設定して親子関係を作りましょう。
    下の「◧ 反転」「🔗 同期」ボタンを使えば、左右対称の腕などを一瞬で作れます。
  • アニメーション: 画面下の「🎬 Animation」タブで、コマごとにREC(録画)して動きを作ります。
  • AIロジック: 「🧠 Logic / AI」タブで、「もし HPが低い なら 逃げる」などの性格を作ります。

🎨 UI Design (画面作り)

タイトル画面やゲーム中のHUD、メニュー、ゲームオーバー画面を作ります。
要素を配置したら、必ず右側の「シーン」が合っているか確認してください。

  • 紐付け変数 (Data Bind): これを設定すると、ゲーム中の「HP」や「時間」と自動で連動します。
  • アクション: ボタンを押した時の動作（ゲーム開始、ジャンプ、攻撃など）を設定します。
  • イベント用画面: 「チュートリアル」用の画面を作りたい場合は、シーンを任意の名前（例：tutorial_1）に変更して作成し、ステージ上の「イベント発火」ギミックから呼び出します。
  • ★ オプション設定画面: 「🎚️スライダー」を配置し、紐付け変数で「設定: BGM音量」などを選ぶと、ゲーム内で音量やFPSを調整できるメニューが作れます。「キー割り当て変更」アクションを使えばキーコンフィグも実装可能です！

💾 保存とゲームの書き出し

💾 保存 (.json) 「編集の続き」をするためのセーブデータです。
次回「📂 開く」から読み込んで編集を再開できます。
📤 書き出し (.html) 「ゲームとして遊ぶ」ための完成ファイルです。
このHTMLファイルを友達に送れば、ブラウザですぐに遊べます！
※このファイルから編集の続きはできません。
📥 素材読込 3Dモデル(GLB/GLTF)や、画像をテクスチャとして読み込めます。
